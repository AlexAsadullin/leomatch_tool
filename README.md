# leodv

Автоматизация просмотра анкет в `@leomatchbot` (Telegram). Скачивает медиа, дедуплицирует анкеты, поддерживает авто-режимы (авто-дизлайк / авто-лайк), ротацию нескольких аккаунтов и ведёт галерею с оценками. Управлять можно через браузер или прямо через Telegram-бот.

## Что нужно перед началом

- **Python 3.13** (проверить: `python3 --version`)
- **uv** — быстрый менеджер пакетов: `pip install uv` или [документация](https://github.com/astral-sh/uv)
- Один или несколько аккаунтов Telegram с номером телефона
- Telegram API-ключи (получить бесплатно на [my.telegram.org/apps](https://my.telegram.org/apps))

### Как получить TG_API_ID и TG_API_HASH

1. Зайди на [my.telegram.org/apps](https://my.telegram.org/apps) — войди своим аккаунтом
2. Нажми «Create application», заполни любое название и платформу (например, Desktop)
3. Скопируй `App api_id` (число) и `App api_hash` (строка) — они понадобятся в `.env`

## Первый запуск — пошагово

### Шаг 1. Скачать проект

```bash
git clone <repo-url>
cd leodv
```

### Шаг 2. Создать виртуальное окружение и установить зависимости

```bash
uv venv venv/
source venv/bin/activate       # macOS / Linux
# venv\Scripts\activate        # Windows

uv pip install -r requirements.txt
```

### Шаг 3. Создать `.env`

```bash
cp .env.example .env
```

Открой `.env` в любом редакторе и заполни:

```ini
TG_API_ID=12345678             # число с my.telegram.org
TG_API_HASH=abcdef1234567890   # строка с my.telegram.org

# Телефоны аккаунтов через запятую (международный формат).
# Порядок = порядок ротации при исчерпании лимита.
TG_PHONES=+79991234567,+79998765432

SESSION_NAME=leodv             # префикс .session-файлов, можно оставить

TG_BOT_TOKEN=                  # токен Telegram-бота (опционально, см. ниже)
```

> Файл `.env` содержит секреты — никогда не коммить его в git.

### Шаг 4. Авторизовать Telegram-аккаунты

```bash
python3 login.py
```

Для каждого номера из `TG_PHONES` скрипт попросит код из SMS (или из самого Telegram). После успешной авторизации в корне появятся файлы `leodv_<digits>.session`.

При повторном запуске уже авторизованные аккаунты пропускаются — скрипт безопасно запускать снова.

### Шаг 5. Запустить приложение

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

В консоли должно появиться:
```
INFO: Telethon connected; bootstrapping latest message
INFO: Application startup complete.
```

Открой в браузере: **[http://127.0.0.1:8000](http://127.0.0.1:8000)**

> `--reload` автоматически перезапускает сервер при изменении кода. Для продакшна убери этот флаг.

## Ежедневное использование

Каждый раз достаточно одной команды:

```bash
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Интерфейс автоматически обновляется каждую секунду — просто оставь вкладку браузера открытой.

### Что происходит после запуска

1. Приложение подключается к `@leomatchbot` через Telethon
2. Загружает последнюю анкету (если она уже была)
3. Ждёт новых сообщений от бота

Когда бот присылает анкету — фото/видео скачиваются, анкета проверяется на дубли и отображается в интерфейсе. Нажимаешь ❤️ или 👎 — реакция уходит боту.

## Режимы работы

Все режимы переключаются кнопками на главном экране (или в Telegram-боте).

| Режим | Что делает |
|---|---|
| **Авто-дизлайк** | Все входящие анкеты автоматически получают 👎, без показа |
| **Авто-лайк** | Все входящие анкеты автоматически получают ❤️, уведомление приходит в TG-бот |
| **Только новые** | Анкеты, которые уже показывались раньше (дубли), получают 👎 автоматически |
| **Авто-смена акков** | При достижении лимита аккаунт меняется автоматически, без остановки |
| **Фильтр возраста** | Анкеты вне диапазона (например, 18–25) → 👎 автоматически |

> Авто-лайк и авто-дизлайк взаимоисключающие — включить оба нельзя.

### Порядок применения фильтров

При каждой входящей анкете проверки идут строго в этом порядке:

1. Совпадение с `highest_priority.json` → показать сразу, игнорируя все остальные фильтры
2. Совпадение с `auto_skip.json` → 👎 без показа
3. Авто-дизлайк / только новые / фильтр возраста → 👎 без показа
4. Авто-лайк → ❤️ без показа
5. Показать пользователю

## Лимиты аккаунтов

`@leomatchbot` ограничивает количество действий в сутки. Приложение отслеживает два вида лимитов:

**Лимит от Telegram** — бот присылает сообщение «Слишком много… за сегодня». Обрабатывается автоматически.

**Встроенные мягкие лимиты** — когда наш счётчик достигает порогов:

| Действие | Порог |
|---|---|
| Авто-дизлайков с одного аккаунта | 1 400 |
| Авто-лайков с одного аккаунта | 30 |

При достижении порога:
- Если **Авто-смена акков** включена — переключается на следующий аккаунт автоматически
- Если выключена — интерфейс блокируется с кнопкой ручной смены
- Если лимиты исчерпаны на **всех** аккаунтах — программа останавливается, в Telegram-бот приходит статистика и сообщение об остановке

Счётчики сбрасываются раз в сутки (в 05:00 по местному времени).

## Фильтры: авто-скип и приоритет

Два JSON-файла в папке `app/`:

### `app/auto_skip.json`

Список подстрок описания. Если описание анкеты содержит хотя бы одну из них — автоматически 👎.

```json
[
  "не ищу серьёзного",
  "только дружба"
]
```

### `app/highest_priority.json`

Список подстрок. Если описание совпадает — анкета показывается немедленно, игнорируя все авто-режимы и фильтры (кроме авто-скипа).

```json
[
  "Маша, 22, Москва"
]
```

Оба файла создаются автоматически при первом запуске (пустые списки). Редактируй в любом текстовом редакторе — изменения применяются при следующей анкете (перезапуск не нужен).

## Telegram-бот (опционально)

Позволяет управлять приложением прямо из Telegram — удобно когда браузер недоступен.

### Настройка

1. Создай бота через [@BotFather](https://t.me/BotFather) — получишь токен вида `123456:ABC-DEF...`
2. Добавь токен в `.env`:
   ```ini
   TG_BOT_TOKEN=123456:ABC-DEF...
   ```
3. Перезапусти приложение

При старте бот автоматически находит все аккаунты из `TG_PHONES` и открывает им доступ.

### Возможности бота

- Получать анкеты пушем (фото + описание)
- Ставить ❤️ / 👎 / 💌 инлайн-кнопками прямо под анкетой
- Переключать все режимы (авто-лайк, авто-дизлайк, и т.д.)
- Менять фильтр возраста
- Переключить аккаунт вручную
- Кнопка **ℹ️ Статус** — таблица статистики по каждому аккаунту

## Скрипты

Запускать из корня проекта с активированным venv.

### `login.py`

Авторизация аккаунтов из `TG_PHONES`. Запускать один раз при первом старте или после истечения сессии.

```bash
python3 login.py
```

### `scripts/download_tg_media.py`

Скачать все сторисы канала/пользователя по ссылке.

```bash
python3 scripts/download_tg_media.py https://t.me/username/s/16
python3 scripts/download_tg_media.py https://t.me/username/s/16 /путь/к/папке
```

Скачивает активные, закреплённые и архивные сторисы. Если файл уже скачан — сохраняет новую копию с суффиксом `_2`, `_3` и т.д. Использует аккаунты из `TG_PHONES`.

### `scripts/extend_db.py`

Разовая утилита для наполнения БД из истории чата `@leomatchbot`. Нужна только при первоначальной миграции.

### `backup.py`

Ручной бэкап `data.db`:

```bash
python3 backup.py
```

## HTTP API

### Управление

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/state` | снапшот AppState + `total_profiles` + `active_phone` |
| POST | `/api/like` | отправить ❤️ |
| POST | `/api/dislike` | отправить 👎 |
| POST | `/api/letter` | `{"text":""}` — отправить 💌 / 📹 с опциональным текстом |
| POST | `/api/switch-account` | сменить аккаунт вручную |
| POST | `/api/auto-dislike/toggle` | вкл/выкл авто-дизлайк |
| POST | `/api/auto-like/toggle` | вкл/выкл авто-лайк |
| POST | `/api/only-new/toggle` | вкл/выкл «только новые» |
| POST | `/api/auto-rotate/toggle` | вкл/выкл авто-смену аккаунта |
| POST | `/api/age-filter` | `{"min":18,"max":25}` или `{"min":null,"max":null}` для сброса |

### Галерея

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/gallery` | список анкет: id, description, rating, seen_count, thumb_url |
| GET | `/api/ratings/{id}` | анкета + все медиа из архива |
| POST | `/api/ratings/{id}/rate` | `{"rating": 1-10}` |

Статика: `/media/*` → `media/`, `/static/*` → `static/`.

## Технические детали

### База данных

Таблица `profiles` в `data.db`:

| Колонка | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | |
| `description` | TEXT | полный текст описания |
| `first_media_hash` | TEXT | sha256 первого фото или ffmpeg-хеш первого кадра видео |
| `seen_count` | INTEGER | сколько раз анкета пришла от бота |
| `first_seen_at` / `last_seen_at` | TEXT | ISO-timestamps |
| `registered_at` | INTEGER | unix-timestamp |
| `rating` | INTEGER | оценка 1–10, -1 = не оценена |

`db.init()` применяет миграции через `ALTER TABLE ... ADD COLUMN` — БД никогда не дропается.

### Дедупликация

Анкета считается дублем если совпадает **хотя бы одно** из:
- `canonicalize(description)` — NFKC + casefold + удаление zero-width символов + collapse whitespace
- sha256 любого из фото альбома совпадает с `first_media_hash` в БД

Оба индекса хранятся в памяти, строятся при старте из полной таблицы.

### Медиа-архив

При каждой анкете (новой или дубле) медиа **копируются** в `media/_archive/{profile_id}/` — идемпотентно, существующие файлы пропускаются. Этот каталог никогда не очищается автоматически.

Текущая анкета хранится в `media/{profile_id}/` и удаляется при появлении следующей.

### Watchdog

Фоновая задача проверяет каждые 30 сек: если бот молчит и нет текущей анкеты — забирает последнее сообщение из истории и прогоняет через пайплайн. Защита от пропущенных событий.

### Соглашения

- Весь Telethon-I/O — только через `app/tg.py`
- `state.lock` — единственный мьютекс, все мутации `AppState` под ним
- `profile.py` — единственный модуль, одновременно работающий с `db`, `dedup`, `tg`, `state`
- Авто-режимы проверяются в строгом порядке: priority → auto_skip → авто-фильтры → показ пользователю
