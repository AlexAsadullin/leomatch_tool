# leodv

Автоматизация просмотра анкет в `@leomatchbot` с дедупликацией, фильтрами и переключением аккаунтов. Два независимых интерфейса управления:

- **🌐 Веб-фронт** — браузер по `http://127.0.0.1:8000`.
- **🤖 Telegram-бот** — отдельный бот с reply-клавиатурой прямо в Telegram.

Оба интерфейса используют общий стейт — можно лайкать с веба, потом переключаться на бота и продолжать с того же места.

---

## Установка

```bash
source venv/bin/activate
pip3 install -r requirements.txt
cp .env.example .env
# отредактировать .env (см. ниже)
python3 login.py             # авторизация юзер-аккаунтов (TG_PHONES)
```

Минимальный `.env`:
```ini
TG_API_ID=…                  # https://my.telegram.org/apps
TG_API_HASH=…
TG_PHONES=+79991234567,+79998765432,+79997654321
# для бота — опционально, см. ниже
TG_BOT_TOKEN=
```

---

## Запуск

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Это **единственная команда** на запуск. Внутри одного процесса поднимаются:
- юзер-клиенты Telethon для `TG_PHONES`,
- веб-сервер FastAPI (фронт),
- Telegram-бот (если задан `TG_BOT_TOKEN`).

---

## 🌐 Вариант 1 — веб-фронт

Открыть в браузере: **http://127.0.0.1:8000**

Что есть:
- большая карточка с фото и описанием текущей анкеты;
- кнопки **❤️ Лайк / 👎 Дизлайк / 💌 / 📹**;
- поле ввода своего письма + галка «авто-сообщение»;
- режимы: **Авто-лайк, Авто-дизлайк, Только новые** (тоггл-кнопки);
- фильтр возраста (форма min/max);
- **Переключить аккаунт** — циклическая ротация по `TG_PHONES`;
- баннеры приоритетных анкет и предупреждений (лимит, не-анкета и т.д.);
- статистика лайков/дизлайков, активный телефон.

Состояние опрашивается каждую секунду — изменения с бота видны почти сразу.

## 🤖 Вариант 2 — Telegram-бот

### Когда удобнее
- управление с телефона без браузера;
- получать новые анкеты пушем в Telegram;
- никаких портов, доступ к внешнему IP, HTTPS и пр. — бот сам коннектится в Telegram.

### Настройка
1. У `@BotFather` создать нового бота → получить **TOKEN**.
2. В `.env`:
   ```ini
   TG_BOT_TOKEN=12345:AA…
   ```
3. Запустить `uvicorn …`.

Всё. Никаких `ADMIN_IDS` руками — при старте бот сам делает `/start` от лица
каждого `TG_PHONES` через их юзер-клиенты. Это (а) даёт боту реальные `user_id`
владельцев, (б) кеширует `InputPeer` в его sqlite-сессии — без этого Telegram
не даёт боту отправлять файлы тем, кто с ним не переписывался.

В логах при старте появится строка вроде `Bot admin_ids после bootstrap: [...]`.

### Как пользоваться

Все кнопки — на **reply-клавиатуре снизу экрана** (не inline под сообщением). Состояние режимов написано прямо на кнопке:

```
[ ❤️ Лайк ]                 [ 👎 Дизлайк ]
[ 💌 / 📹 авто ]            [ 💌 / 📹 своё ]
[ Авто-лайк: ON|OFF ]       [ Авто-дизлайк: ON|OFF ]
[ Только новые: ON|OFF ]
[ Возраст: 18-25 | — ]
[ 🔄 Переключить аккаунт ]  [ ℹ️ Статус ]
```

- **Анкеты** приходят сообщением: альбом фото + подпись с `id`, `seen_count`, описанием. Приоритетные анкеты помечены `⚠️ ПРИОРИТЕТНАЯ АНКЕТА!`.
- **Лайк/Дизлайк** — нажми кнопку, реакция уходит сразу.
- **💌 / 📹 авто** — отправит письмо с автотекстом по возрасту анкеты.
- **💌 / 📹 своё** — бот спросит текст → пришли его одним сообщением → отправит.
- **Возраст** — нажми → бот спросит формат → пришли `18-25` (или `off` для отключения).
- **Авто-лайк / Авто-дизлайк / Только новые** — одно нажатие переключает (текст на кнопке обновится при следующем сообщении).
- **Переключить аккаунт** — циклически берёт следующий из `TG_PHONES`, делает kickoff `"1"` к `@leomatchbot` → анкеты идут с нового аккаунта.
- **ℹ️ Статус** — текущая статистика, активный телефон, флаги.
- **Статусные сообщения** (`«Меняю аккаунт…»`, `«Отправляю письмо…»`, лимит исчерпан) — приходят отдельным сообщением в чат.

Чужие user'ы (не из `TG_PHONES`) получат «Доступ запрещён» — команды не выполняются.

## Какой интерфейс выбрать

| | Веб | Telegram-бот |
|---|---|---|
| Откуда работать | браузер | приложение Telegram |
| Свежие анкеты | сами появляются в карточке | приходят пушем |
| Управление с мобильного | работает, но мелковато | удобно |
| Запуск без TG-бота | да, веб всегда есть | требует `TG_BOT_TOKEN` |
| Сетевые требования | localhost / порт 8000 | только исходящий MTProto |

Можно держать оба одновременно — они синхронизированы через общий стейт. Например: открыл веб для просмотра, а лайки/переключения делаешь с телефона из бота.

Если `TG_BOT_TOKEN` пустой — бот выключен, остаётся только веб.

## Скрипты (`scripts/`)

- `scripts/extend_db.py` — однократная заливка истории чата `@leomatchbot` в БД (для дедупликации). Использует чекпоинт `scripts/db_extension_info.json`, переживает прерывания. Аккаунты для парсинга — константа `ACCOUNTS` в файле.
- `scripts/find_duplicates.py` — отчёт по уже существующим дублям в БД (SQLAlchemy).
- `scripts/claude_rest.py` — служебное (генерация `.claude/settings.json` из `.gitignore`).

Запускать из корня проекта: `python3 scripts/<script>.py`.

---

## Архитектура коротко

- **`app/main.py`** — FastAPI lifespan: поднимает `db`, `dedup`, юзер-клиенты, опционально бот.
- **`app/tg.py`** — обёртка Telethon для юзер-аккаунтов (`TG_PHONES`), ротация.
- **`app/profile.py`** — пайплайн обработки анкет, авто-режимы, фильтры.
- **`app/dedup.py`** — in-memory индексы дедупликации (canonical text + photo hashes).
- **`app/state.py`** — общий `AppState`, доступен веб- и бот-интерфейсам.
- **`app/db.py`** — SQLite (raw sqlite3).
- **`static/`** — веб-фронт (vanilla JS).
- **`tg_bot/bot.py`** — Telegram-бот с reply-клавиатурой.

`app/highest_priority.json` — массив подстрок описания приоритетных анкет (matches тех, что нужно показывать всегда, даже сквозь фильтры).
